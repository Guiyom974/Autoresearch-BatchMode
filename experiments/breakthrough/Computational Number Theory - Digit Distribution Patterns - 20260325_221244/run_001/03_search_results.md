### 1. Wikipedia: Markov Chain
A Markov chain (or Markov process) is a stochastic model describing a sequence of possible events where the probability of each event depends solely on the state attained in the previous event [[1]](https://en.wikipedia.org/wiki/Markov_chain). This "memoryless" characteristic is known as the Markov property [[1]](https://en.wikipedia.org/wiki/Markov_chain). The changes of state within the system are called transitions, and the probabilities associated with these changes are called transition probabilities [[1]](https://en.wikipedia.org/wiki/Markov_chain). If the state space is finite, these probabilities can be represented by a transition matrix, where each row sums to one [[1]](https://en.wikipedia.org/wiki/Markov_chain). Markov chains are widely used in statistical modeling, economics, biology, and signal processing [[1]](https://en.wikipedia.org/wiki/Markov_chain).
**Source:** [Wikipedia - Markov chain](https://en.wikipedia.org/wiki/Markov_chain)

### 2. dida: What is a Markov Chain?
This article explains Markov chains using a practical weather prediction example (Sunny, Cloudy, Rainy) [[2]](https://dida.do/what-is-a-markov-chain). By knowing the current weather state, you can predict the next day's weather using transition probabilities without needing the historical sequence of previous days [[2]](https://dida.do/what-is-a-markov-chain). To represent this efficiently, a square transition matrix is used, where rows represent the current state, columns represent the next state, and each cell contains the probability of transitioning between them [[2]](https://dida.do/what-is-a-markov-chain). 
**Source:** [dida - What is a Markov Chain?](https://dida.do/what-is-a-markov-chain)

### 3. Brilliant Math & Science Wiki: Markov Chains
Brilliant defines a Markov chain as a mathematical system that transitions from one state to another according to fixed probabilistic rules [[3]](https://brilliant.org/wiki/markov-chains/). The state space can be anything from numbers and letters to baseball scores and stock performances [[3]](https://brilliant.org/wiki/markov-chains/). The article highlights key properties of states within a Markov chain, such as whether a state is *absorbing* (once entered, it cannot be left), *recurrent* (the system will eventually return to it), or *transient* [[3]](https://brilliant.org/wiki/markov-chains/). 
**Source:** [Brilliant - Markov Chains](https://brilliant.org/wiki/markov-chains/)

### 4. University of Auckland: Chapter 8 - Markov Chains
This academic text provides a formal mathematical definition of Markov chains and transition matrices [[4]](https://www.stat.auckland.ac.nz/~fewster/325/notes/ch8.pdf). It explains that a transition matrix $P$ must list all possible states in a state space $S$, and the matrix is always an $N \times N$ square matrix [[4]](https://www.stat.auckland.ac.nz/~fewster/325/notes/ch8.pdf). The element $p_{ij}$ represents the probability of making a transition *from* state $i$ *to* state $j$ in a single step [[4]](https://www.stat.auckland.ac.nz/~fewster/325/notes/ch8.pdf). The text also covers how to calculate multi-step transition probabilities using matrix multiplication [[4]](https://www.stat.auckland.ac.nz/~fewster/325/notes/ch8.pdf).
**Source:** [Stat.auckland.ac.nz - Chapter 8: Markov Chains](https://www.stat.auckland.ac.nz/~fewster/325/notes/ch8.pdf)

### 2. MARCOS: Deep Thinking by Markov Chain of Continuous Thoughts (2025)
**Summary:** In the realm of Large Language Models (LLMs), generating discrete Chain-of-Thought (CoT) reasoning can be slow and forces models to "think and speak" simultaneously. This paper introduces MARCOS, a novel paradigm that models reasoning as a Markov chain of *continuous* thoughts. By decoupling the thinking and speaking stages and modeling the randomness in reasoning, MARCOS allows for explicit control over the reasoning process. Experiments on benchmarks like GSM8K show that MARCOS significantly outperforms existing continuous reasoning baselines in accuracy while providing up to a 15.7× speedup compared to traditional token-based CoT [[2]](https://arxiv.org/pdf/2509.25020).
