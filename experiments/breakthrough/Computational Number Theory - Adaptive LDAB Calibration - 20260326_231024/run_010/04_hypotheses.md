**Hypotheses for the “Methodological Validation and Correction of Primorial Gap Variance Extraction”**  
Below are five testable hypotheses that directly address the three research questions, build on the prior failure of weighting‑only approaches, and respect the computational constraints ( k ≤ 10 ).  Each hypothesis is stated, its testability is justified, and a concrete experiment is outlined.

| # | Hypothesis (statement) | Why it is testable | Experiment that would test it |
|---|------------------------|--------------------|--------------------------------|
| **1** | **Invariant‑correctness of the gap extractor** – For every primorial index k (3 ≤ k ≤ 10) the newly designed algorithm will output exactly φ(Pₖ) gaps, and the sum of those gaps will equal Pₖ. | The claim is a pure logical invariant that can be verified by a deterministic program: count the number of output gaps and compute their sum. No statistical inference is required. | *Implementation*: write a gap‑extraction routine that (a) generates the reduced residue system modulo Pₖ via a wheel sieve, (b) sorts the residues, (c) forms consecutive differences, and (d) records the count and total.  Run the routine for all k = 3,…,10 and assert (using an internal test harness) that **count = φ(Pₖ)** and **sum = Pₖ**.  Any violation flags a bug in the extraction logic. |
| **2** | **Non‑zero, monotonic trend of the empirical variance** – Once the full gap set is correctly extracted, the population variance σ²(Pₖ) is strictly positive for each k ≥ 3 and grows monotonically (i.e., σ²(Pₖ₊₁) > σ²(Pₖ)). | σ² can be computed exactly from the validated gap list (a finite set of integers).  Because the gap distribution widens as the modulus gets larger, we expect a deterministic trend that can be observed by direct calculation. | Using the same validated gap sets from Hypothesis 1, compute the population variance  \(\sigma^2 = \frac{1}{\phi(P_k)}\sum_{i=1}^{\phi(P_k)} (g_i - \mu)^2\) where \(\mu = P_k / \phi(P_k)\).  Plot σ² versus k (or versus log Pₖ) and check (a) σ² > 0 for all k, (b) the sequence is monotonic (or at least non‑decreasing).  The test is a straightforward numeric check; no probabilistic modelling is needed. |
| **3** | **Agreement with a modified Cramér random‑model baseline** – The empirically observed variance σ²(Pₖ) will fall within ±5 % of the variance predicted by a Cramér‑type random model that respects the primorial coprimality constraints (i.e., the gaps are generated as if φ(Pₖ) random points were placed uniformly on a circle of circumference Pₖ). | Theoretical variance for a uniform random placement of φ points on a circle of length Pₖ is known analytically:  \(\sigma^2_{\text{cramér}} = \frac{P_k^2}{12\,\phi(P_k)}\) (the variance of spacings in a Poisson‑process of rate φ/Pₖ).  We can compare this to the empirical σ². | (i) Derive the closed‑form expression above.  (ii) For each k, compute the relative error  \(\Delta = \frac{|\sigma^2_{\text{emp}} - \sigma^2_{\text{cramér}}|}{\sigma^2_{\text{cramér}}}\).  (iii) Verify that Δ < 0.05 for all k ≤ 10.  If the bound is violated, the hypothesis is false, indicating a systematic deviation from the random‑spacing assumption. |
| **4** | **Power‑law scaling of the variance with primorial size** – The empirical variance follows a scaling law of the form  \(\sigma^2(P_k) \sim C \cdot P_k^{\alpha}\) (or equivalently \(\sigma^2 \sim C' \cdot (\log P_k)^{\beta}\)) for some exponent α (or β) that can be estimated by regression. | Once we have the numeric σ² values for k = 3,…,10, we can fit both a power‑law model (log σ² versus log Pₖ) and a logarithmic model (log σ² versus log log Pₖ) using ordinary least squares.  The goodness‑of‑fit (R²) will tell us which model is plausible. | Perform a log‑log linear regression of σ² on Pₖ (or on log Pₖ) for the 8 data points.  Compute the estimated exponent α̂ (or β̂) and the 95 % confidence interval.  Also compute the coefficient of determination R².  If R² > 0.95 and the residuals appear randomly scattered, the power‑law (or log‑log) hypothesis is supported.  If not, consider alternative scaling forms (e.g., σ² ∝ φ(Pₖ)). |
| **5** | **High‑precision arithmetic eliminates underflow artifacts** – When computing σ² for the largest admissible primorial (P₁₀ ≈ 6.4 × 10⁹) using arbitrary‑precision rational arithmetic (or a sufficiently high‑precision decimal library), the resulting variance will be numerically stable and will differ from the double‑precision result by less than 10⁻⁶ relative error. | The hypothesis directly contrasts two arithmetic regimes (double vs. high‑precision) on the same deterministic data.  By re‑running the variance calculation with a high‑precision library (e.g., Python’s `decimal` with 50 digits, or `mpmath` with 100‑digit precision) we can quantify the discrepancy. | (i) Implement the gap‑extraction and variance calculation using standard IEEE‑754 double precision.  Record σ²_double.  (ii) Re‑implement the same algorithm using `mpmath.mp` with 100‑digit precision (or `fractions.Fraction` for exact rational arithmetic).  Record σ²_high.  (iii) Compute the relative error  \(\epsilon = |\sigma^2_{\text{high}} - \sigma^2_{\text{double}}| / \sigma^2_{\text{high}}\).  If ε < 10⁻⁶, the hypothesis is confirmed; otherwise, the double result suffers from underflow/overflow and the high‑precision route is required for k ≥ 10. |

### How These Hypotheses Advance the Research

| Research Question | Addressed by |
|-------------------|--------------|
| **Algorithmic Correctness** | Hypothesis 1 guarantees the extractor produces the correct, complete set of gaps (the previous “gaps = 2” bug is directly refuted). |
| **True Empirical Variance** | Hypothesis 2 yields the validated variance values and reveals their trend, providing the “verified empirical baseline” called for in the problem statement. |
| **Theoretical Baseline Comparison** | Hypothesis 3 places the empirical numbers against a rigorous random‑spacing model, answering whether the observed variance is merely a statistical artifact or reflects genuine structure. |
| **Scaling Behaviour** | Hypothesis 4 offers a quantitative scaling law, enabling future extrapolation and testing of higher‑order LDAB models. |
| **Numerical Stability** | Hypothesis 5 resolves the floating‑point underflow problem highlighted in prior runs, ensuring that variance differentials remain meaningful for the largest feasible primorials. |

Together, these hypotheses form a coherent validation pipeline: **(1) correct data → (2) correct metric → (3) correct comparison → (4) correct scaling model → (5) correct numerical precision**.  Each is independently testable, uses the same computational infrastructure (optimised prime sieves, wheel factorization, high‑precision libraries), and stays within the stipulated limit of k ≤ 10.  Once all five are confirmed, the research will have a solid, reproducible foundation for re‑introducing logarithmic scaling and LDAB calibration studies.
