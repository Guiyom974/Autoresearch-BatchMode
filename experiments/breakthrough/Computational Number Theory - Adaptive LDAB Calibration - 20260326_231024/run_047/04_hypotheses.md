**3‑5 Testable Hypotheses for the “Arbitrary‑Precision Framework for LDAB Calibration at High Primorial Indices ( k > 10 )”**

| # | Hypothesis (statement) | Why it is testable | Experiment that would test it |
|---|------------------------|-------------------|--------------------------------|
| **1** | **Precision‑scaling law** – For k > 10 the minimum mantissa length (in bits) needed to avoid overflow/NaN in the LDAB binomial‑coefficient and log‑density calculations grows roughly **linearly with the logarithm of the primorial**, i.e.  \[P_{\min}(k)=\alpha\;\log_2 P_k+\beta\quad(k>10),\] where α ≈ 0.2–0.3 (to be refined by data). | The relationship can be expressed as a simple mathematical function of observable quantities (k, Pₖ). By measuring the smallest precision that yields a finite result for each k we obtain a data set that can be fitted with ordinary least‑squares. | 1. Implement the LDAB pipeline in **gmpy2/mpmath** with a *dynamic precision* routine that starts at the smallest possible bit‑count (e.g. 64 bits) and increments by 32‑bit blocks until the computation finishes without Inf/NaN.  <br>2. Record the first successful bit‑count for each k = 11, 12, … , 20.  <br>3. Plot \(P_{\min}(k)\) versus \(\log_2 P_k\) and fit the linear model to obtain α and β. |
| **2** | **Computational‑overhead scaling** – The wall‑clock time for a single high‑k LDAB update scales **super‑linearly (≈ quadratic) with the number of precision bits** used, because the underlying big‑integer arithmetic (multiplication, factorial, binomial) dominates the cost. | Runtime is a directly observable, quantitative variable that can be recorded for many combinations of (k, bits). If the scaling is truly quadratic, a log‑log plot of time versus bits will be linear with slope ≈ 2. | 1. For a fixed k = 12 (the first “critical” index) run the arbitrary‑precision pipeline with precision = 128, 256, 512, 1024, 2048 bits.  <br>2. Measure elapsed time (using `time.perf_counter()`) for each run (average over ≥ 20 repetitions to reduce variance).  <br>3. Plot \(\log T\) versus \(\log\) bits and test the slope (H₀: slope ≤ 1.5, H₁: slope > 1.8).  <br>4. Repeat for k = 15 to see whether the exponent stays constant. |
| **3** | **KL‑divergence achievability** – With **≥ 512‑bit mantissa** the high‑precision LDAB pipeline can keep the KL divergence between the modeled prime density and the empirical (or known) prime count **below 10⁻⁴** for every primorial index up to k = 15. | KL divergence is a well‑defined scalar metric; it can be computed exactly from the LDAB output probabilities and the ground‑truth distribution (e.g., π(x) for x = Pₖ). If the metric stays under the threshold, the hypothesis is confirmed; if any k produces a larger value, the hypothesis is falsified. | 1. Run the 512‑bit arbitrary‑precision LDAB pipeline for k = 11,…,15.  <br>2. Compute the KL divergence \(D_{\mathrm{KL}}(p_{\text{LDAB}}\|p_{\text{true}})\) for each k.  <br>3. Compare the maximum observed Dₖ with the 10⁻⁴ target.  <br>4. If the target is met, the hypothesis holds; if not, lower the precision (e.g., 384 bits) and repeat to locate the *critical* precision. |
| **4** | **Minimal‑precision “phase‑transition”** – For each k > 10 there exists a sharp **precision threshold** \(P_{\text{c}}(k)\) such that any mantissa length below \(P_{\text{c}}(k)\) yields overflow/NaN, while any length ≥ \(P_{\text{c}}(k)\) yields stable finite results. The function \(P_{\text{c}}(k)\) follows the same linear‑log law as hypothesis 1, but with a larger intercept (the “safety margin”). | By performing a binary‑search on the precision for each k, we can pinpoint the exact cut‑off where the output switches from non‑finite to finite. The existence of a clear transition is an empirical fact that can be verified statistically (e.g., using a Kolmogorov‑Smirnov test on the distribution of outcomes across many random seeds). | 1. For each k = 11,…,20, use a **bisection search** on the bit‑count: start with low = 64 bits, high = 2048 bits.  <br>2. At each step compute the LDAB output and record whether it is finite.  <br>3. Continue until the interval width ≤ 8 bits; the midpoint is taken as \(P_{\text{c}}(k)\).  <br>4. Plot the collected \((k,P_{\text{c}}(k))\) and test the linear‑log relationship (hypothesis 1) with a separate offset parameter. |
| **5** | **Diminishing returns of extra precision** – Once the KL divergence is below 10⁻⁴, **doubling the mantissa length beyond the minimal stable precision does not reduce the KL divergence by more than a factor of 1.2** (i.e., < 20 % improvement). | This is a classic *performance‑accuracy trade‑off* statement that can be tested by measuring the KL divergence at several multiples of the minimal precision (e.g., 1×, 2×, 4×). The hypothesis is falsified if any doubling yields > 20 % reduction. | 1. Determine \(P_{\text{c}}(k)\) for a representative set of k (e.g., 11, 13, 15) using the binary‑search method from hypothesis 4.  <br>2. Run the LDAB pipeline at \(P_{\text{c}}(k)\), \(2P_{\text{c}}(k)\), and \(4P_{\text{c}}(k)\).  <br>3. Compute KL divergence for each run.  <br>4. Compute the improvement ratio \(\rho = D_{\text{original}}/D_{\text{doubled}}\).  If all ρ < 1.2, the hypothesis holds; otherwise it is rejected. |

### How the Hypotheses Build on Prior Findings

* **run_027** showed that VMR checks work up to k = 8 and highlighted the need for larger‑k checks.  
  → Hypothesis 1 directly addresses the *precision* needed to keep those checks valid beyond k = 8.

* **run_033** observed a **deviation of the VMR scaling exponent** from the assumed 0.80 for k ≥ 8.  
  → Hypothesis 4 posits that the deviation may be an *artifact of insufficient precision*; by locating the true precision‑threshold we can re‑evaluate the exponent.

* **run_036** identified the generic double‑precision overflow at k ≈ 132 (log P > 709).  
  → Hypothesis 5 predicts that **LDAB overflow occurs much earlier** (k ≈ 11‑12) because the combinatorial terms in the density estimate explode faster than the simple exponential in the generic overflow case. The experiment in Hypothesis 1 will confirm or refute this earlier overflow point.

Together, the five hypotheses cover the three core research questions (precision scaling, computational overhead, high‑index stability) and provide concrete, measurable predictions that can be evaluated with the proposed arbitrary‑precision implementation.
